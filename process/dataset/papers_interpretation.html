1 研究背景：推测解码 Speculative Decoding 原理与挑战

1.1 为什么需要推测解码：自回归解码的三重瓶颈

大语言模型的标准推理方式是逐 token 自回归生成—— 每生成一个新 token xt+1，都必须完整执行目标模型一次前向传播。 这种串行性带来三重根本性瓶颈：

⚠️ 内存带宽瓶颈

每次前向传播需将数百 GB 模型权重从 HBM 加载到计算单元， 而每次仅产出 1 个 token。 解码阶段是典型的内存带宽受限（memory-bound）场景， 大部分时间花在搬数据而非计算上。

⚠️ 串行依赖 + GPU 利用率极低

xt+1 依赖 xt，token 生成无法并行化。 单序列推理（batch=1）时算术强度极低， GPU 利用率不足 10%—— 大量计算单元闲置，却不得不等待内存搬运完成。

💡 关键洞察：Transformer 的并行验证特性

Transformer 有一个重要特性——单次前向传播处理 k 个 token 的延迟 ≈ 处理 1 个 token 的延迟 （在内存带宽瓶颈区域，延迟主要由权重加载决定，与输入长度几乎无关）。 如果能提前猜出后续若干 token，让大模型一次性并行验证， 就能在一次前向调用中产出多个 token——这就是推测解码的核心动机。

随着 OpenAI o1、DeepSeek-R1 等长思维链模型兴起， 推理输出动辄数千 token，解码延迟已成为端到端延迟的主要来源。 "在不改变输出分布的前提下加速解码"因此成为最具工程价值的方向之一。

1.2 推测解码：基本框架与两阶段流程

推测解码（Speculative Decoding）由 Leviathan et al. (2023) 和 Chen et al. (2023) 同时提出， 是当前最主流的无损加速范式。核心思路是"小模型起草 + 大模型验证"：

  • 草稿模型 (Draft Model, q)：轻量级小模型，速度快，快速猜出未来 γ 个 token
  • 目标模型 (Target Model, p)：大模型，速度慢但质量高，一次前向并行验证所有草稿 token
推测解码标准流程
① 草稿阶段

小模型（草稿器）自回归生成 \(\gamma\) 个候选 token， 同时记录每步概率分布 \(q_1, \ldots, q_\gamma\)。 延迟 \(T_{\text{draft}} \propto \gamma\)。

② 验证阶段

大模型对 [前缀, 草稿 token] 做一次前向， 得到每位置概率分布 \(p_1, \ldots, p_\gamma\)。 延迟 ≈ 生成 1 个 token 的开销。

③ 接受 / 拒绝

逐 token 按拒绝采样判断：接受概率 = \(\min(1, p(x_k)/q(x_k))\)。 保留最长一致前缀，首个拒绝处丢弃后续全部 token。

④ 产出 token

本轮产出 \(1 \sim \gamma\!+\!1\) 个 token（含 bonus token）。 加速比 \(\eta = \tau / (1 + T_{\text{draft}}/T_{\text{verify}})\)， \(\tau\) 为平均接受长度。

关键在于：验证阶段只花一次大模型前向的开销（与直接生成 1 个 token 相同）， 却可能一次产出多个 token。只要草稿质量足够好，加速就天然成立。

\[ L = T_{\text{draft}} + T_{\text{verify}} = T_{\text{draft}} + L_{\text{target}} \] 加速比 \(\eta = L_{\text{target}} \big/ \bigl(T_{\text{draft}} + L_{\text{target}}\bigr)\)，\(\tau\) 为平均接受长度

要让 η 变大，只有两条路：提升平均接受长度 τ（草稿更准）， 或压缩草稿开销 Tdraft（草稿更快）。后续所有工作都在围绕这两个变量做文章。

1.3 经典算法详解：拒绝采样 Rejection Sampling

推测解码的精妙之处在于其接受/拒绝机制—— 基于修正拒绝采样（Modified Rejection Sampling）， 保证无论草稿模型 q 质量如何，最终输出分布精确等于目标模型 p 的分布。

算法流程

给定已生成前缀，每轮执行：

📌 拒绝采样算法（γ 个草稿 token）

Phase 1 — 草稿：小模型 \(q\) 自回归生成 \(\gamma\) 个候选 token \(\tilde{x}_1, \tilde{x}_2, \ldots, \tilde{x}_\gamma\)， 同时记录每步的概率分布 \(q_1, q_2, \ldots, q_\gamma\)。

Phase 2 — 验证：大模型 \(p\) 对 \([\text{前缀},\, \tilde{x}_1, \ldots, \tilde{x}_\gamma]\) 做一次前向， 得到每个位置的概率分布 \(p_1, p_2, \ldots, p_\gamma, p_{\gamma+1}\)。

Phase 3 — 逐 token 接受/拒绝：对第 \(i\) 个草稿 token \(\tilde{x}_i\)：

  • 计算接受概率 \(\alpha_i = \min\!\left(1,\; \dfrac{p_i(\tilde{x}_i)}{q_i(\tilde{x}_i)}\right)\)
  • 以 \(\alpha_i\) 概率接受 \(\tilde{x}_i\)，继续检查下一个
  • 若拒绝，从残差分布 \(p'(x) = \text{normalize}\!\bigl(\max(0,\, p_i(x) - q_i(x))\bigr)\) 采样一个 token 作为替代，本轮结束

Bonus token：若全部 \(\gamma\) 个草稿都被接受，额外从 \(p_{\gamma+1}\) 采样一个 token 作为"奖励"。

💡 接受概率的直觉

\(\alpha_i = \min(1,\, p/q)\) 的含义： 当目标模型 \(p\) 比草稿模型 \(q\) 更倾向于这个 token 时（\(p > q\)），一定接受； 当 \(p < q\) 时（草稿模型"过度自信"），以 \(p/q\) 的概率接受—— 等于把草稿模型"多分配"的概率按比例收回。

单位置期望接受率 \(\alpha = \displaystyle\sum_{x} \min\bigl(p(x),\, q(x)\bigr) = 1 - D_{\text{TV}}(p, q)\)， 即两个分布的重叠面积。\(p\) 和 \(q\) 越相似，\(\alpha\) 越接近 1，加速效果越好。

1.4 无损保证：为什么输出分布严格等于目标模型

这是推测解码最重要的性质。以下证明（单 token 情形）展示其无损性：

输出某个 token \(t\) 有两种来源：

情况一：接受草稿 token（草稿模型采样到 t 且被接受）

\[ P(\text{accept } t) \;=\; q(t) \cdot \min\!\left(1,\; \frac{p(t)}{q(t)}\right) \;=\; \min\bigl(q(t),\; p(t)\bigr) \]

情况二：拒绝后从残差分布重采样

\[ P(\text{resample } t) = \frac{\Bigl[1 - \displaystyle\sum_{x} \min\bigl(p(x),\, q(x)\bigr)\Bigr] \cdot \max\bigl(0,\, p(t) - q(t)\bigr)}{1 - \displaystyle\sum_{x} \min\bigl(p(x),\, q(x)\bigr)} \] \[ = \max\bigl(0,\; p(t) - q(t)\bigr) \]

合并两种情况：

\[ P(\text{output} = t) \;=\; \min\bigl(q(t),\, p(t)\bigr) + \max\bigl(0,\, p(t) - q(t)\bigr) \;=\; p(t) \;\;\checkmark \]
✅ 核心结论

利用恒等式 \(\min(a,b) + \max(0,\, a-b) = a\)， 最终输出分布精确等于目标模型 p 的分布。 无论草稿模型 q 质量如何——哪怕 q 很差——输出分布都不会偏移。 草稿质量只影响加速比（接受率），不影响输出正确性。 这就是"无损加速"的严格数学保证。

📌 理论加速比

设单位置接受概率为 \(\alpha\)，草稿长度为 \(\gamma\)，则一轮中期望产出 token 数：

\[ \mathbb{E}[\text{tokens per round}] \;=\; \frac{1 - \alpha^{\,\gamma+1}}{1 - \alpha} \]

理想极限（草稿开销 \(\ll\) 验证开销，\(\gamma \to \infty\)）：最大加速比 \(= \dfrac{1}{1-\alpha}\)。

1.5 树形推测解码：从单链到多分支

经典推测解码的草稿是一条线性序列（单链）， 一旦某个 token 被拒绝，后续所有草稿 token 全部作废。 树形推测解码将候选组织为树形结构， 在单次大模型前向中验证多条路径。

线性草稿
  • 候选路径数：1 条
  • 验证方式：逐 token 串行判断
  • 首个拒绝点后：全部作废
树形草稿
  • 候选路径数：多条（指数级扩展）
  • 验证方式：树注意力单次前向并行验证
  • 某个分支被拒：其他分支仍可接受

树注意力 Tree Attention

树形推测解码的关键技术是树注意力（Tree Attention）： 将草稿树展平为序列输入目标模型， 使用特殊的祖先-后代注意力掩码—— 每个节点只能 attend 到自己的祖先节点，非同一路径的 token 之间互不可见。 这样目标模型的一次前向就能同时验证树上所有候选 token。

代表性方法

Medusa（ICML 2024）

在目标模型上附加 K 个预测头，并行预测未来第 1, 2, ..., K 位置的 token。 候选树以笛卡尔积形式构建：每个头预测 top-k 个候选组合成树。 静态预定义树结构，与上下文无关。

EAGLE / EAGLE-2

EAGLE 引入特征级自回归 + 树形草稿。EAGLE-2 进一步提出动态草稿树—— 用草稿模型置信度近似接受率，动态生成树结构： 高置信度路径深扩，低置信度路径早停，避免资源浪费。 这是上下文感知的树形推测解码。

2 EAGLE-3：从特征预测到直接 Token 预测的跃迁

2.1 问题定位与解决思路

EAGLE-3 针对的是 EAGLE 系列推测解码中一个被长期忽视的根本性问题： 增加训练数据无法持续提升加速比——数据扩展定律不成立。 这意味着无论投入多少训练资源，EAGLE 的加速比都会"触顶"，无法像大模型本身那样从数据扩展中受益。

⚠️ EAGLE-3 要解决的问题

大模型社区的核心趋势是扩展训练数据来提升能力（不增加推理成本）。 但对 EAGLE 来说，数据量翻 8 倍，接受长度几乎没有变化——扩展定律失效。 这使推测解码无法搭上"数据扩展"的便车，成为其进一步发展的瓶颈。

EAGLE-3 的解决角度是从根源上找约束： 为什么 EAGLE 无法从数据中学习更多？因为草稿模型被特征预测约束锁死了—— 它必须拟合目标模型的顶层特征，这限制了表达能力。 同时，仅用顶层特征丢弃了多层语义信息。 两项改进分别对应这两个瓶颈：

以往工作忽略的问题
  • 特征预测是不必要的：EAGLE 要求草稿输出拟合顶层特征，但这只是一种设计选择，并非推测解码的本质要求
  • 仅用顶层特征是浪费：目标模型不同层捕获不同粒度信息，但 EAGLE 只取最高层
  • 训练-推理不一致：训练时用真实特征，推理时用预测特征，误差累积
EAGLE-3 的解决角度
  • 去除特征预测：草稿模型直接预测 token（通过 Training-time Test），释放表达能力
  • 多层特征融合：低、中、高三层特征拼接+FC 降维，信息更丰富
  • 训练时模拟推理：在训练中执行"测试步骤"，让模型适应自身输出，消除不一致
🎯 核心洞察

特征预测约束是 EAGLE 设计中的一个历史包袱—— 它在 EAGLE v1 中用于让草稿模型在特征空间做自回归（因为特征空间比 token 空间更平滑）， 但这个约束同时也锁死了模型从更多数据中学习的潜力。 去掉它之后，数据扩展定律自然成立，峰值加速达到 6.5×。

2.2 EAGLE 系列演进：从特征自回归到直接预测

要理解 EAGLE-3 的贡献，需要先回顾 EAGLE 系列的演进脉络。 三代工作一脉相承，每一代都在前一代的基础上解决一个关键瓶颈。

EAGLE（v1）：特征级自回归

核心思想是在特征空间而非 token 空间做自回归。 草稿模型接收目标模型的顶层特征，预测下一个特征，再通过目标模型的 LM head 得到 token。 特征空间比 token 空间更平滑，预测更容易。同时引入上一步采样结果作为输入， 解决 token 层采样带来的不确定性。采用树形草稿 + 树注意力并行验证。

EAGLE-2：动态草稿树

EAGLE v1 的草稿树结构是静态预定义的，与上下文无关。 EAGLE-2 用草稿模型的置信度近似接受率，动态生成草稿树结构， 在 drafting 阶段末尾做剪枝，避免资源浪费。这是上下文感知的树形推测解码。

EAGLE-3：去除特征预测 + 多层融合

发现 EAGLE 的特征预测约束阻碍了数据扩展。 通过 Training-time Test 直接预测 token，用多层特征融合替代仅用顶层特征， 首次实现了"训练数据越多 → 加速比越高"的扩展定律。

2.3 核心问题：为什么 EAGLE 无法从数据扩展中受益？

大模型社区的一个趋势是扩展训练数据来提升模型能力，而不增加推理成本。 然而论文作者观察到：对 EAGLE 来说，增加训练数据的收益非常有限—— 数据量翻 8 倍，接受长度几乎没有提升（见 Figure 1 中 EAGLE-2 的平坦曲线）。

⚠️ 两大瓶颈

瓶颈一：特征预测约束限制了表达能力。 EAGLE 要求草稿模型的输出必须拟合目标模型的顶层特征 ft+1， 这意味着草稿模型被限制在目标模型特征空间的一个子空间中，无法充分发挥自身潜力。 约束越紧，模型越难从更多数据中学习。

瓶颈二：仅用顶层特征丢失了多层信息。 目标模型的不同层捕获不同粒度的语义：低层关注语法/词法，中层关注句法/局部语义，高层关注全局语义。 EAGLE 只用顶层特征，等于扔掉了低层和中层的丰富信息。

2.4 核心改进一：Training-time Test——去除特征预测

从"预测特征"到"预测 token"

EAGLE 的草稿模型工作流程是： 输入特征 → 预测下一特征 f̂t+1 → 通过 LM head 得到 token t̂t+1。 训练时有两个损失：特征预测损失 lfea（让 f̂t+1 ≈ ft+1）和 token 预测损失 ltok。

EAGLE-3 的改变是：彻底去掉特征预测损失 lfea， 草稿模型直接输出一个不受约束的向量 at+1， 再通过 LM head 得到 token。at+1 不需要拟合任何特定目标，模型获得了完全自由。

EAGLE：特征预测

输入 f1, f2, ···, ft（目标模型顶层特征）

→ 草稿模型预测 f̂t+1 ≈ ft+1

→ LM head 得到 t̂t+1

  • 损失 = lfea + ltok
  • 输出必须拟合顶层特征
  • 输入必须用顶层特征
  • 表达能力受限
EAGLE-3：直接 token 预测

输入 g1, g2, ···, gt（多层融合特征）

→ 草稿模型输出 at+1（无约束向量）

→ LM head 得到 t̂t+1

  • 损失 = 仅 ltok
  • 输出不受任何约束
  • 输入可用任意层特征
  • 表达能力完全释放
📌 举例：为什么要去掉特征预测？

想象你在学画画（草稿模型），目标是画出一幅和大师（目标模型）一样的画：

  • EAGLE 的方式：老师要求你必须先用大师的笔法（特征）画草稿， 再由大师润色成成品。你的笔法被限制在大师的风格里， 即使你学了更多画（更多训练数据），也只能在大师的框架内发挥。
  • EAGLE-3 的方式：老师只看最终成品像不像（token 预测损失）， 你可以用任何笔法画草稿。现在你学得越多，画得越像——数据扩展定律生效了。

多步生成的训练时模拟

去除特征预测后带来了一个新问题：多步草稿生成时，第 2 步以后的输入怎么办？

在推理时，草稿模型需要连续生成多个 token。第 1 步可以用目标模型的特征作为输入， 但第 2 步需要第 1 步生成的 token 对应的特征——而此时第 1 步的 token 还没有被目标模型验证， 无法获得目标模型的特征。EAGLE-3 的解决方案是：用草稿模型自己的输出 a 来替代。

但这引入了训练-推理不一致：训练时草稿模型总是看到目标模型的真实特征（"教师强制"）， 而推理时只能看到自己的近似输出（"自由运行"）。EAGLE-3 通过 Training-time Test 解决这个问题： 在训练过程中模拟推理时的多步生成——生成 a 后将其喂回模型做下一步训练， 让模型在训练时就习惯自己的输出作为输入。

💡 关键细节：对角线注意力掩码

在模拟步骤中，草稿模型自己生成的 token（如 aI）与原始训练数据之间 没有正常的因果依赖关系——"I" 是草稿模型从 "How can" 预测出来的， 而非训练数据中原有的。因此注意力掩码从下三角矩阵变为对角线矩阵： 草稿预测只 attend 到自己对应的位置，不与其他位置交互。

由于对角线掩码下矩阵乘法会浪费大量计算（大部分位置不参与）， EAGLE-3 改用向量点积替代矩阵乘法，仅计算对应位置的注意力分数，效率更高。

2.5 核心改进二：多层特征融合

EAGLE 仅使用目标模型的顶层特征作为草稿模型的输入。 EAGLE-3 认为不同层捕获不同粒度的信息，应该全部利用：

  • 低层特征 l：词法、语法等局部信息
  • 中层特征 m：句法、局部语义
  • 高层特征 h：全局语义、任务理解

具体做法：从目标模型分别取出低、中、高三个层的特征序列（每个维度为 k，即隐藏层大小）， 将三者拼接为 3k 维向量，经过一个全连接层（FC）降维回 k 维， 得到融合特征 g。g 同时包含了三个层次的信息。

\[ g = \text{FC}\!\Bigl(\bigl[\, \mathbf{l} \oplus \mathbf{m} \oplus \mathbf{h} \,\bigr]\Bigr) \] \(\mathbf{l}, \mathbf{m}, \mathbf{h} \in \mathbb{R}^k\) → 拼接为 \(3k\) 维 → FC 降维 → \(g \in \mathbb{R}^k\)
📌 类比：多层特征融合

把目标模型想象成一个翻译公司，有三层员工：

  • 低层（初级翻译）：精通语法，能把每个词翻对
  • 中层（资深翻译）：理解句法，能处理长难句
  • 高层（翻译总监）：把握全局语境，确保译文符合目标文化

EAGLE 只问翻译总监（顶层特征），得到的是高层概括，细节可能缺失。 EAGLE-3 把三层员工的意见融合在一起，草稿模型同时获得语法、句法和语义信息， 预测更准确。

EAGLE-3 的实验设计直接回答了三个核心问题：

❓
问题一：数据扩展定律是否成立？

→ 是。 Figure 1 清楚显示：数据量从 1× → 8×，EAGLE-2 的接受长度几乎不变（~4.0 → ~4.2）， 而 EAGLE-3 持续上升（~4.5 → ~5.5+）。扩展定律从"不成立"变为"成立"， 直接验证了去除特征预测约束的核心价值。

❓
问题二：两项改进各自贡献多少？

→ 消融实验（表 2）逐步叠加：EAGLE-2 基线 3.16× → +去除特征预测 3.82×（+0.66×）→ +多层融合 4.40×（+0.58×）。 两项改进均有显著独立贡献，且去除特征预测贡献更大—— 这验证了"特征预测是不必要约束"这一核心假设。

❓

2.11 结论与遗留问题

EAGLE-3 核心贡献

EAGLE-3 通过两项关键改进——Training-time Test（去除特征预测）和 多层特征融合——实现了推测解码领域的重大突破：

  • 加速比：最高 6.5×，比 EAGLE-2 快约 1.4×
  • 扩展定律：首次证明训练数据增加可持续提升加速比
  • 生产可用：在 SGLang / vLLM 中大 batch size 下仍有效
  • 无损保证：输出分布与目标模型完全一致
⚠️ EAGLE-3 的遗留问题（后续工作的切入点）

EAGLE-3 虽然解决了数据扩展问题，但仍有两个结构性局限， 成为 DFlash 和 DDTree 的出发点：

局限一：输入层融合导致深层信息稀释。 EAGLE-3 把融合特征与 token embedding 拼接后送入草稿模型输入层， 但随着层数加深，目标信息会被逐层稀释—— 这就是为什么 EAGLE-3 只能用 1 层草稿模型。 DFlash 的 KV 注入正是解决此问题。

局限二：自回归草稿的串行延迟。 EAGLE-3 仍采用自回归方式逐 token 生成草稿， γ 步草稿需要 γ 次串行前向。DFlash 的块扩散并行生成解决了此问题。

🔗 作为后续工作的基线

EAGLE-3 是 DFlash 和 DDTree 的直接对比基线。 理解 EAGLE-3 的设计选择和上述局限性，是理解后续工作动机的关键—— DFlash 针对"信息稀释 + 串行延迟"提出 KV 注入 + 块扩散， DDTree 则在 DFlash 基础上进一步用草稿树替代单链验证。

"EAGLE-3 removes the feature prediction constraint, instead directly predicting draft tokens through a Training-time test. Second, it replaces the use of the target model's top-layer features with a fusion of the target model's lower, middle, and upper-layer features to obtain richer information." —— EAGLE-3 论文结论

3 Paper 1 · DFlash：块扩散驱动的并行草稿

3.1 问题定位：EAGLE-3 的两个遗留瓶颈

DFlash 针对的是 EAGLE-3 虽然解决了数据扩展问题后仍存在的两个结构性瓶颈 （详见第 2.11 节遗留问题）：

瓶颈一：深层信息稀释

EAGLE-3 在输入层把融合特征与 token embedding 拼接， 但随着草稿模型层数加深，目标信息会被逐层稀释。 这迫使 EAGLE-3 只能使用 1 层草稿模型—— 想加深却不敢加深，草稿质量被结构性限制。

瓶颈二：草稿串行延迟

EAGLE-3 仍采用自回归方式逐 token 生成草稿， γ 个草稿 token 需要 γ 次串行前向传播。 草稿延迟随长度线性增长：Tdraft = γ · tstep。 要控制延迟就只能减少 γ，但 γ 少了接受长度也少。

DFlash 的解决角度是用扩散模型替代自回归， 同时改变特征注入方式——两项创新分别对应两个瓶颈：

方案一：KV 注入（解决信息稀释）

不把目标特征塞进输入层，而是注入草稿模型每一层的 K/V 投影。 目标知识写入 KV 缓存，所有层、所有去噪迭代共用同一份条件信息。 草稿模型可加深到 5 层而不稀释。

方案二：块扩散并行（解决串行延迟）

用块扩散去噪替代自回归：一次前向同时生成整块 token。 Tdraft = tparallel（与块大小无关）。 5 层草稿模型的 wall-clock ≈ 1 层自回归草稿，打破了"深度 vs 速度"的 trade-off。

🎯 核心洞察

目标模型最了解未来。 大型 LLM 的隐藏特征隐式包含多步未来 token 的预测信息。 DFlash 把这些特征作为持久条件（而非一次性输入）， 让轻量级块扩散模型并行预测下一段 token 块—— 草稿模型本质上变成了目标模型深层语义的"扩散适配器"。

⚠️ 以往工作忽略的问题

EAGLE-3 将目标特征视为一次性输入——拼接进输入层就不管了。 但 Transformer 的深层表示会逐渐"遗忘"输入层的额外信息， 这是一种结构性缺陷而非调参问题。 DFlash 首次指出：要充分利用目标模型知识，必须让条件信息贯穿草稿模型的每一层。

3.3 核心创新一：KV 注入 KV Injection

KV 注入是 DFlash 区别于以往工作的关键。 现有方法 EAGLE-3 把目标模型的隐藏特征与草稿模型的 token embedding 在输入层融合， 但随着草稿模型层数加深，目标信息会被逐层稀释， 导致接受长度的边际增益迅速消失。

💡 DFlash 的根本不同

把目标上下文特征视为持久条件信息， 直接注入草稿模型每一层的 K/V 投影中。 投影后的特征写入 KV 缓存，整轮所有去噪迭代共用同一份目标知识。 这让草稿模型在第 1 层与第 5 层接收到的目标条件强度完全一致， 接受长度可以随层数持续增长而不饱和。

📌 类比：KV 注入 vs 输入融合

把草稿模型想象成一位需要猜老师下一句话的学生：

  • 输入融合 / EAGLE-3 风格：开场把老师的笔记瞥了一眼就拿走， 之后就只能凭记忆答题，越往后忘得越多，答案质量逐层下降。
  • KV 注入 / DFlash 风格：把老师的核心笔记贴在课桌每个角落， 无论思考到第几步都能随时参考，答案质量始终有保障。

这就是 DFlash 能用 5 层草稿模型而不会遇到信息稀释的根本原因。

EAGLE-3：输入层融合

目标特征 + 草稿 embedding → 一次性塞进输入

  • 深层信息逐渐稀释
  • 层数加深收益快速递减
  • 特征只在输入端使用一次
  • 实际上仅能用 1 层草稿模型
DFlash：每层 KV 注入

目标特征 → 注入草稿模型每层 K/V 投影

  • 每层都获得强条件信号
  • 接受长度随层数有效扩展
  • 特征在 KV 缓存中持久复用
  • 草稿模型可加深到 5 层及以上

3.5 训练策略

① 多层上下文特征提取

从目标模型中选取 5 个隐藏层抽取隐藏状态， 拼接后通过轻量投影层融合为紧凑的目标上下文特征。 消融实验表明 5 层（5-H 设置）一致优于 3 层（3-H 设置）， 因为多层选取能同时覆盖词法、语法、语义等多个抽象层级，比单层更稳健。

② 损失衰减 Loss Decay

推测解码中早期位置错误会让后续 token 全部失效。 DFlash 用指数衰减权重强调前面位置：

\[ w_k = \exp\!\left(-\,\frac{k-1}{\gamma}\right), \quad \gamma \text{ 控制衰减速率} \]
📌 数值示例：损失衰减的强度

取 γ = 5、块大小 16，各位置相对权重为：

位置 1 = 1.000  |  位置 2 = 0.819  |  位置 3 = 0.670  |  位置 5 = 0.449  |  位置 8 = 0.247  |  位置 16 = 0.050

位置 1 的权重是位置 16 的 20 倍， 模型被强烈引导优先学好前面 token，因为前面的正确率对接受长度影响最大。

③ 随机锚点采样

训练时不再均匀划分序列，而是随机采样锚点 token 来构建 mask 块。 这种简单的数据增强让模型见到更多样的对齐情况，实测能让接受长度再涨约 10%。

④ 共享 Embedding 与 LM Head

草稿模型与目标模型共享词嵌入层和语言建模头，并冻结这部分参数； 只训练草稿 Transformer 主体。 这一方面减少了可训练参数，另一方面把草稿模型显式定位为目标模型表示空间下的"轻量扩散适配器"。

⑤ 训练数据

论文从 NVIDIA Nemotron Post-Training Dataset V2 与 CodeAlpaca 混合采样约 800K 条样本， 关键是不直接使用原始数据，而是用目标模型重新生成响应， 以获得更紧密的目标对齐。 消融实验表明 100K 样本即可达到良好性能，训练成本可控。

→ 消融实验显示移除目标特征条件化后加速比降至 2-3×， 证明 KV 注入是 DFlash 的核心组件，而非锦上添花。

❓
问题四：高并发生产环境是否有效？

→ SGLang 部署中并发=32 仍保持 2.8× 正加速， 而 EAGLE-3(60) 在高并发下跌到 baseline 以下。 并行草稿在大 batch 下优势更明显—— 因为自回归草稿的串行开销在高并发时被放大。

3.8 结论与遗留问题

DFlash 核心贡献

  • ✦ 首次将块扩散模型有效引入推测解码，实现真正意义上的并行草稿生成。
  • ✦ 提出 KV 注入机制，让目标模型的语义条件穿透草稿模型每一层，突破层数瓶颈。
  • ✦ 在 Qwen3-4B / 8B / Coder-30B-A3B 及 LLaMA-3.1-8B 上实现 峰值 6.1×、平均 ~5× 无损加速，全面优于 EAGLE-3。
  • ✦ DFlash(16) 甚至超越 EAGLE-3(60)，更少草稿 token、更长接受长度。
  • ✦ 在长 CoT 推理与高并发部署等真实场景下均保持稳定加速。
  • ✦ 训练成本低，100K 样本即可达到良好效果。
⚠️ DFlash 的遗留问题（DDTree 的切入点）

DFlash 虽然解决了信息稀释和串行延迟两个问题，但仍有一个关键瓶颈：

单链验证的拒绝浪费。 DFlash 的草稿是一条线性轨迹—— 逐 token 接受/拒绝，一旦中间某个 token 被目标模型拒绝， 后续所有草稿 token 全部作废，哪怕它们恰好正确。 这限制了接受长度的进一步提升。

DDTree 的洞察是：块扩散一次前向已经输出了每个位置的完整概率分布， 与其只取 top-1 拼成单链，不如利用这些分布构建草稿树， 让目标模型一次前向验证多条候选路径。

4 Paper 2 · DDTree：从因子化分布到最优草稿树

4.1 问题定位：DFlash 单链验证的拒绝浪费

DDTree 针对的是 DFlash 虽然解决了信息稀释和串行延迟后仍存在的最后一个瓶颈 （详见第 3.8 节遗留问题）：

⚠️ DFlash 的遗留问题

DFlash 的草稿是一条线性轨迹—— 逐 token 接受/拒绝，一旦中间某个 token 被目标模型拒绝， 后续所有草稿 token 全部作废，哪怕它们恰好正确。 这成为接受长度进一步提升的最大瓶颈。

⚠️ 以往工作忽略的问题

DFlash（以及 EAGLE 系列）的草稿模型一次前向实际上输出了每个位置的完整概率分布， 但验证时只取每个位置的 top-1 token 拼成单链—— 剩余的分布信息被完全丢弃。 这些被丢弃的分布恰好可以用来构建多分支草稿树， 为每个位置提供"第二次机会"。

此外，EAGLE-2 虽然也使用树形草稿，但其树结构依赖动态置信度生成—— 需要多次串行前向才能扩展树的深度。 DDTree 则利用块扩散的因子化分布，一次前向就能获得所有位置的概率， 构树几乎零额外开销。

DDTree 的解决角度是：把被丢弃的分布变成草稿树。 块扩散一次前向产出的逐位置边缘分布就是构建草稿树的天然原料。 在固定节点预算 B 下，用 best-first 堆算法选出"最值得验证"的前 B 条前缀， 再用 ancestor-only 注意力让目标模型一次前向并行验证整棵树。

📌 直观对比：单链 vs 草稿树

设目标模型实际生成 "The cat sat on the mat"：

DFlash 单链：草稿 = ["The", "cat", "ran", "fast", "away"]

→ 第 3 个 token "ran" 与目标不符被拒绝， 后续 "fast"、"away" 全部作废。本轮接受长度仅 2。

DDTree 草稿树：同一位置同时给出多个候选：

关键观察：构建草稿树几乎不增加草稿模型开销—— 块扩散模型一次前向就已经吐出每个位置的完整概率分布， 多取几个候选只是在 logits 上多排序而已。

4.2 关键概念：因子化分布 vs 自回归条件分布

这是理解 DDTree 的数学起点，也是论文最容易让读者卡住的地方。 先把两种分布的形式写下来：

目标模型（自回归条件分布）

\[ p(y_{1{:}L} \mid c, b) = \prod_{i=1}^{L} p\!\left(y_i \,\Big|\, c,\, b,\, y_{1{:}i-1}\right) \]

块扩散草稿（因子化边缘分布）

\[ Q(y_{1{:}L} \mid c, b) = \prod_{i=1}^{L} q_i(y_i \mid c, b) \]
📌 直观区分两种分布

设我们要预测两个 token，分别记作位置 1 和位置 2：

  • 条件分布 / 自回归：先抽位置 1 的具体 token，比如 "The"， 位置 2 的概率分布会依赖于 "The" 这个具体选择； 如果位置 1 抽的是 "A"，位置 2 的分布就完全不一样。
  • 因子化分布 / 块扩散：位置 2 的预测不依赖于位置 1 具体抽到了哪个 token， 它只依赖共享上下文 (c, b)。模型同时输出两个位置的边缘分布，互相独立。

看似缺少了条件依赖是块扩散的弱点， 但 DDTree 把它变成优势： 因为各位置独立，可以在每个位置自由组合 top-K token 形成各种候选路径， 天然适合用来构造草稿树。

4.4 代理目标 Surrogate Objective

理想情况下我们要最大化目标模型条件下的期望接受长度， 但目标模型的条件分布 p(yi | c, b, y1:i−1) 在构树时根本拿不到。 DDTree 因此退而求其次，转而优化草稿模型因子化近似下的期望接受长度作为代理：

\[ \sigma(\mathcal{T}) = \mathbb{E}_{Y \sim Q}\!\Bigl[\,\alpha_{\mathcal{T}}(Y)\,\Bigr] = \sum_{u \in \mathcal{T}} Q(u \mid c, b) = \sum_{u \in \mathcal{T}} \prod_{i} q_i(u_i \mid c, b) \]

论文的 Proposition 1 证明： 这个代理目标可以拆解成"每个节点的前缀概率之和"的形式—— 也就是说目标对节点是可加的， 每个节点的贡献彼此独立。

📌 代理目标如何运作

设 L=3 个位置，每位置 top-2：

各候选前缀的代理得分 = 各位置边缘概率的乘积：

The→cat→sat = 0.6 × 0.5 × 0.4 = 0.12

当节点预算 B = 4，选择最高得分的 4 条前缀。 注意 "The"（0.6）和 "The→cat"（0.30）等较短前缀也会被纳入—— 保证树的前缀闭合性。

📖 Proposition 2：top-B 前缀自然形成有效草稿树

把所有非空前缀按 q(u | c, b) 降序排列， 取前 B 条得到 TB。 论文证明 TB 满足前缀闭合， 并且在所有节点数 ≤ B 的有效草稿树中代理目标最大。 直觉是：任何后代的前缀概率严格小于其祖先， 因此降序排序时祖先必然先出现， top-B 集合天然就是一棵有效的树。

4.5 算法 1：best-first 堆构树

由于词表 |V| 巨大，把所有可能前缀显式枚举是不可行的。 好在 Lemma 1 把搜索空间收窄到了"每层只看 top-K = min(B, |V|) 个 token"， 论文提出的 best-first 堆算法在这个空间内按概率降序高效地枚举出前 B 条前缀：

5.1 问题与挑战：并行草稿的结构性瓶颈

并行草稿器（如 DFlash）通过一次前向生成整块草稿 token， 使草稿延迟 \(T_{\text{draft}}\) 几乎与块大小 \(\gamma\) 无关。 但当块大小增大后，两个深层瓶颈浮现——一个在生成质量，一个在系统效率。

瓶颈一：后缀衰减（Suffix Decay）

并行草稿器独立预测每个位置，无法建模块内 token 间的依赖关系。 当上下文存在多种合理续写（如 "of course" 和 "no problem"）时， 模型可能生成不连贯的组合（如 "of problem"），因为每个位置对所有可能的前驱取边际而非条件概率。

→ 接受率沿块快速衰减，后续 token 大量被拒，草稿与验证算力同时浪费。

瓶颈二：验证浪费（Verification Waste）

确定最优验证长度是难题。并行生成能轻松产出长草稿块， 但不加区分地验证全部 token在高并发场景下严重拖垮吞吐—— 每个被拒绝的 token 都占用了本可服务其他请求的批处理容量。

→ 理想验证长度随数据侧（代码高、聊天低）和系统侧（轻载低代价、重载高代价）双轴变化， 静态策略无法适配。

回顾推测解码的每 token 延迟公式：

\[ L = \frac{T_{\text{draft}} + T_{\text{verify}}}{\tau} \]

自回归草稿器（如 EAGLE-3）以 \(T_{\text{draft}} \propto \gamma\) 为代价换取高 \(\tau\)； 并行草稿器（如 DFlash）把 \(T_{\text{draft}}\) 压到一次前向但 \(\tau\) 受后缀衰减拖累； 同时固定长度验证浪费 \(T_{\text{verify}}\)。 DSpark 同时从提升 \(\tau\) 和降低有效 \(T_{\text{verify}}\) 两个杠杆入手。

这看似反直觉：逐步自回归理应比并行更准确。但论文的逐位置分析揭示了根本原因—— 位置 1 的杠杆效应。推测解码是严格前缀匹配：位置 1 被拒则整块作废。 而两种架构在位置 1 的差异纯粹来自架构容量：

  • 自回归草稿器因 \(T_{\text{draft}} \propto \gamma\) 必须用浅层网络（1 层），位置 1 容量受限
  • 并行草稿器 \(T_{\text{draft}} \approx O(1)\) 可用深层网络（5 层），位置 1 显著更准

例如在 Chat 上，DFlash 位置 1 接受率 0.72 vs Eagle3 仅 0.53—— 这个初始差距不成比例地放大到最终接受长度。 DSpark 的设计目标：保留并行骨干的高初始容量，同时用轻量串行头补偿后缀衰减。

块分布分解为逐位置条件概率的乘积：

\[ P(\mathbf{X} \mid x_0) = \prod_{k=1}^{\gamma} p_k(x_k \mid x_0, x_{\lt k}) \]

每个位置的条件概率由并行骨干的基础 logits与串行头的转移偏置共同决定：

\[ p_k(v \mid x_0, x_{\lt k}) = \frac{\exp\!\big(U_k(v) + B_k(x_0, x_{\lt k}, v)\big)}{\displaystyle\sum_{u \in \mathcal{V}} \exp\!\big(U_k(u) + B_k(x_0, x_{\lt k}, u)\big)} \]

其中 \(U_k\) 是并行骨干在位置 \(k\) 的基础 logit，\(B_k\) 是串行头基于已采样前缀 \(x_{\lt k}\) 给出的修正。 采样时从左到右逐位置进行，但串行头必须计算轻量（\(T_{\text{seq}} \ll T_{\text{parallel}}\)），使整体延迟仍由并行阶段主导。

串行阶段有两种实例化：

默认 Markov Head

转移偏置仅依赖前一个 token：\(B_k \to B(x_{k-1}, x_k)\)。 完整的 \(V \times V\) 矩阵太大，用低秩分解 \(B = W_1 W_2\)（\(r=256\)）近似：

\[ B(x_{k-1}, \cdot) = W_1[x_{k-1}] \cdot W_2 \in \mathbb{R}^V \]

\(W_1\) 作 embedding 查表，\(W_2\) 作 logit 投影。 存储和逐步计算均很小，即使词表 \(V \approx 10^5\) 也高效。

📌 具体走查：上下文 "Thank you"，并行骨干给出位置 1 基础 logits \(U_1\)， 采样得到 "of"。位置 2 的基础 logits \(U_2\) 对 "course" 和 "problem" 给出相近分数。
→ 无 Markov head：位置 2 边际化所有前驱，"problem"（来自 "no problem" 模式）可能胜出
→ 有 Markov head：\(B(\text{"of"}, \cdot) = W_1[\text{"of"}] \cdot W_2\)， 该向量在 "course" 方向有正值、在 "problem" 方向有负值， 叠加到 \(U_2\) 后 "course" 胜出 → 生成 "of course"
可选 RNN Head

Markov head 是无记忆的——位置 \(k\) 无法访问 \(x_{k-1}\) 之前的 token。 RNN head 维护循环状态 \(s_k\) 累积块内完整前缀历史。

输入 \(z_k = [s_{k-1};\, W_1[x_{k-1}];\, h_k] \in \mathbb{R}^{2r+d}\)，门控更新：

\[ s_k = \sigma(W_g z_k) \odot s_{k-1} + (1 - \sigma(W_g z_k)) \odot \tanh(W_c z_k) \]

\[ B_k(x_{\lt k}, \cdot) = W_2^\top \tanh(W_o z_k) \]

实验显示 RNN head 仅在更长草稿块（\(\gamma \geq 12\)）上有边际增益， 因实现复杂度更高且部署属性不利，默认使用 Markov head。

🔑
少一点自回归，走很远的路

实验表明：2 层 DSpark 即超过 5 层 DFlash（论文 Figure 3）。 仅通过轻量串行头注入局部自回归信息， 就比单纯堆深并行层获得更好的序列连贯性—— 参数效率极高的 accuracy-parameter 权衡。随草稿长度 \(\gamma\) 从 4→16 增大， DSpark 对 DFlash 的优势持续扩大（\(\gamma=7\): +16%/15%/18% → \(\gamma=15\): +30%/26%/22%）， 因为 DFlash 的后缀衰减在长块上更严重，而 DSpark 有效抑制了衰减。

🔍
DSpark vs DDTree：同根不同路

两者都建立在 DFlash 并行草稿之上，但解决不同层面的问题：

  • DDTree保持并行草稿不变，利用其逐位置边缘分布构建多分支草稿树—— 解决"单链拒绝浪费"，侧重验证结构优化
  • DSpark修改并行草稿本身，用串行头注入块内依赖—— 解决"后缀衰减"，侧重草稿质量优化； 同时用置信度调度解决高并发验证浪费，面向生产部署

两者互补：DDTree 在离线 benchmark 上追求峰值加速比（8.2×）， DSpark 在真实生产系统中追求稳健的吞吐-延迟权衡（V4 提速 60%–85%）。

5.2.2 痛点二：验证浪费 → 置信度调度验证

半自回归架构让 DSpark 能高效生成大草稿块， 但产出更多 token 不自动转化为更高端到端加速—— 不加区分地验证全块在高并发下反而拖垮吞吐。 DSpark 用置信度头 + 硬件感知调度器解决。

组件一：置信度头（Confidence Head）

为每个草稿位置输出标量估计 \(c_k \in (0, 1)\)， 建模条件存活概率——给定前缀全部被接受时，位置 \(k\) 的 token 通过目标模型验证的概率：

\[ c_k = \sigma\!\left(\mathbf{w}^\top [h_k;\, W_1[x_{k-1}]\right) \]

其中 \(h_k\) 是并行骨干的隐藏状态，\(W_1[x_{k-1}]\) 是前一草稿 token 的 Markov embedding。 监督信号为解析接受率——由草稿分布与目标分布的总变差距离直接给出：

\[ c_k^* = 1 - \tfrac{1}{2} \| p_k^d - p_k^t \|_1 \]

这个公式直接来自拒绝采样的接受规则：token \(x_k\) 被接受的概率恰为 \(\min(1, p_t(x_k)/p_d(x_k))\)，对草稿分布求期望后恰好等于 \(1 - \frac{1}{2}\|p_d - p_t\|_1\)。 因此最小化分布距离 = 最大化接受率 = 提升存活概率估计。

为什么需要校准？

调度器需要置信度的绝对数值（而非仅排序）来计算期望吞吐 \(\Theta = \tau \cdot \text{SPS}(B)\)。 神经网络通常过度自信——预测 \(c_k = 0.9\) 但实际接受率仅 0.7。 如果直接用过度自信的分数，调度器会高估存活概率，纳入过多本应裁剪的 token，导致次优决策。

DSpark 引入序列温度缩放（Sequential Temperature Scaling, STS）。 关键观察：联合存活概率因式分解为累积乘积 \(\prod_{i \leq k} c_i\)（链式法则）。 STS 在验证集上从左到右逐步校准：对每个位置 \(k\)， 固定已校准的位置 \(1, \ldots, k-1\)，对位置 \(k\) 的温度做 1D 网格搜索， 最小化累积乘积 \(\prod_{i \leq k} c_i\) 的期望校准误差（ECE）。

温度缩放是保序变换（单调），不破坏置信度头学到的相对排序—— 只是把绝对数值"校准"到与经验接受率一致。

📊
校准效果

原始模型 ROC-AUC 0.81–0.90（判别力强）但 ECE 3%–8%（过度自信）； STS 校准后平均 ECE 降至 ~1%，为调度器提供可靠的存活概率估计。

组件二：硬件感知前缀调度器（Hardware-Aware Prefix Scheduler）

以往方法用静态阈值裁剪低置信 token， 但在动态服务环境中，验证一个 token 的代价取决于当前系统负载： 轻载时额外验证几乎免费，重载时占用关键批容量。 DSpark 将验证长度选择形式化为全局吞吐最大化问题。

对于 \(R\) 个活跃请求，设请求 \(r\) 的验证长度为 \(\ell_r\)， 位置 \(j\) 的存活概率为累积乘积 \(a_{r,j} = \prod_{i \leq j} c_{r,i}\)。 批次总 token 数 \(B = \sum_{r=1}^{R}(1 + \ell_r)\)， 期望接受 token 数 \(\tau = \sum_{r=1}^{R}\left(1 + \sum_{j=1}^{\ell_r} a_{r,j}\right)\)。 设引擎吞吐曲线为 \(\text{SPS}(B)\)（初始化时一次性 profiling 的代价表）， 调度器目标为：

\[ \Theta = \tau \cdot \text{SPS}(B) \quad \longrightarrow \quad \max_{\ell_1, \ldots, \ell_R} \]

由于 \(a_{r,j}\) 随 \(j\) 单调不增（越往后存活概率越低）， 全局按 \(a_{r,j}\) 降序排列的贪心准入天然尊重块内前缀依赖—— 位置 \(j\) 的 token 只有在其前缀 \(1, \ldots, j-1\) 全部被纳入后才可能被选中。 调度器沿此贪心路径逐步纳入 token，每次 \(O(1)\) 查表更新 \(\Theta\)， 当吞吐下降时早停。

无损保证：非预期性（Non-anticipating）

无损推测解码有一条铁律：准入决策不得依赖未来候选 token。 如果调度器"偷看"位置 \(k+1\) 的 token 来决定是否纳入位置 \(k\)， 就会引入选择偏差——被纳入的概率不再仅取决于草稿质量， 而是被未来信息扭曲，破坏了拒绝采样的精确性。

DSpark 的早停机制确保无损：截断决策仅依赖当前步已处理的前缀， 当吞吐 \(\Theta\) 下降时立即停止——此刻的决定完全基于已纳入的 token， 不涉及任何尚未处理的未来 token。这保证了目标分布的精确恢复。

5.2.3 训练目标

训练时目标模型冻结，草稿模型共享其 embedding 和 LM head（均冻结）， 仅更新骨干、串行头和置信度头。 三项损失按位置权重 \(w_k = \exp(-(k-1)/\gamma)\) 加权—— 越靠前的位置权重越大，因为推测解码是前缀匹配：位置 1 被拒则整块作废， 前缀位置对期望接受长度的贡献远大于后缀。

① 传统推测解码

6.4 关键原理总结

🔑
目标模型最了解未来： 大模型的隐藏特征已经隐式编码了多步未来 token 的预测信息， 这是 DFlash 高接受长度的根本原因。
🔑
KV 注入 优于 输入融合： 每层都能"参考老师笔记"远比只在开头看一眼更稳定， 草稿模型可以加深而不被信息稀释吞没。
🔑
并行生成 优于 串行生成： 块扩散并行去噪天然贴合 GPU 并行特性， 避免了自回归的串行瓶颈和误差累积。
🔑
树结构 优于 单链： 草稿树为每个位置提供"第二次机会"， 单点拒绝不再触发后续连锁失败。
🔑
前重树 优于 均匀树： 在固定节点预算下，把节点集中到高概率轨迹的前几层， 比平均分配更高效——接受长度对加速比的贡献远高于分支数。
🔑
因子化分布 是树构建的天然基础： 块扩散输出的边缘分布"看似缺少条件依赖"， 但正是这种位置间独立性允许我们灵活组合 top-K token， 让 best-first 堆构出代理意义下的最优树。
🔑
半自回归 兼得 容量与连贯： 并行骨干提供高初始容量（深层网络），轻量 Markov head 注入局部依赖抑制后缀衰减—— 2 层 DSpark 即超 5 层 DFlash，少一点自回归走很远的路。
🔑
硬件感知调度 优于 静态阈值： 验证一个 token 的代价取决于系统负载——轻载时几乎免费，重载时占用关键批容量。 置信度头 + 硬件感知调度器将验证长度形式化为全局吞吐最大化 \(\Theta = \tau \cdot \text{SPS}(B)\)， 轻载榨干空闲算力，重载保护关键容量。

四篇工作的演进脉络

🦅 EAGLE-3：打破扩展瓶颈

问题：EAGLE 系列的"特征预测"约束使草稿模型被锁死在目标模型特征空间的子空间中， 增加训练数据无法持续提升接受率——数据扩展定律不成立。

方案：① 用 Training-time Test 去除特征预测，草稿模型直接预测 token； ② 用多层特征融合（低/中/高三层）替代仅用顶层特征。

效果：扩展定律首次成立，峰值加速 6.5×，比 EAGLE-2 快约 1.4×。 但仍限于 1 层草稿模型（输入层融合的深层稀释）和自回归串行草稿。

⚡ DFlash：并行草稿 + 深层注入

问题：EAGLE-3 的两个遗留瓶颈——① 输入层融合导致深层信息稀释，草稿模型只能用 1 层； ② 自回归草稿 γ 步串行前向，延迟随草稿长度线性增长。

方案：① KV 注入——把目标特征写入草稿模型每一层的 K/V 投影， 信息不再逐层稀释，草稿模型可加深到 5 层；② 块扩散并行——一次前同时去噪整块 token， 5 层草稿的 wall-clock ≈ 1 层自回归草稿。

效果：平均 ~5× 无损加速，全面优于 EAGLE-3（约 2.7 倍）， DFlash(16) 甚至超越 EAGLE-3(60)。

🌳 DDTree：从分布到最优草稿树

问题：DFlash 单链验证——一旦中间 token 被拒，后续全部作废， 而块扩散一次前向输出的逐位置完整分布被丢弃。

方案：用 best-first 堆算法从因子化分布中构建代理最优草稿树（O(B log B)）， 目标模型一次前向 + ancestor-only mask 并行验证整棵树。

效果：60/60 全部场景一致改进，峰值 8.2×， 把"单点拒绝即截断"放宽为"沿替代分支继续接受"。
